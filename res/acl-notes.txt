Виды правил.

* = null
x = id группы, доски, нити или сообщения
- = правило не имеет смысла
+ = правило имеет смысл
g = правило для группы
b = правило для доски
t = правило для нити
p = правило для сообщения

				-t (x, x, x, *)
+b (x, x, *, *)
+t (x, *, x, *)
+g (x, *, *, *)
				-t (*, x, x, *)
+b (*, x, *, *)
+t (*, *, x, *)
				-p (x, x, x, x)
				-p (x, x, *, x)
				-p (x, *, x, x)
+p (x, *, *, x)
				-p (*, x, x, x)
				-p (*, x, *, x)
				-p (*, *, x, x)
+p (*, *, *, x)

Интерпретация разрешений.

Доска:
1) View		- показывать доску в списке досок, разрешить предпросмотр доски
2) Change	- разрешить создание нитей
3) Moderate	- показывать ip, разрешить банить, разрешить изменять настройки нитей и постов

Нить:
1) View		- показывать нить в предпросмотре, показывать нить целиком
2) Change	- разрешить ответы в нить
3) Moderate	- разрешить измнение настроек нити

Сообщение:
1) View		- показывать пост в предпросмотре нити, показывать пост в просмотре нити
2) Change	- не определено
3) Moderate	- не определено

Порядок проверки прав.

Права проверяются от частного к общим.
Примеры:

Права на просмотр сообщения будут проверяться так:
1) Доступно ли для просмотра заданное сообщение для группы пользователя?
- да => сообщение доступно для просмотра, конец
- нет => сообщение не доступно для просмотра, конец
- нет такого правила => переходим к 2)
2) Доступно ли для просмотра заданное сообщение для всех групп?
- да => сообщение доступно для просмотра, конец
- нет => сообщение не доступно для просмотра, конец
- нет такого правила => сообщение доступно для просмотра, конец
Отсутствие каких-либо правил в данном случае означает, что сообщение доступно для просмотра, потому что запрос
на доступность или не доступность просмотра сообщения может быть сделан только если нить доступна для просмотра,
а значит сообщение наследует права на просмотр от нити. Аналогичное утверждение справедливо и для нитей.

Права на просмотр доски будут проверяться так:
1) Доступна ли для просмотра заданная доска для группы пользователя?
- да => доска доступна для просмотра, конец
- нет => доска не доступна для просмотра, конец
- нет такого правила => переходим к 2)
2) Доступна ли для просмотра заданная доска для всех групп?
- да => доска доступна для просмотра, конец
- нет => доска не доступна для просмотра, конец
- нет такого правила => переходим к 3)
3) Доступен ли просмотр группе пользователя?
- да => доска доступна для просмотра, конец
- нет => доска не доступна для просмотра, конец
- нет такого правила => ошибка в таблице acl, не заданы разрешения по умолчанию для группы

Права на редактирование нити будут проверяться так:
1) Доступна ли нить для редактирования группе пользователя?
- да => редактирование нити доступно, конец
- нет => редактирование нити не доступно, конец
- нет такого правила => 2)
2) Доступна ли нить для редактирования всем группам?
- да => редактирование нити доступно, конец
- нет => редактирование нити не доступно, конец
- нет такого правила => 3)
3) Доступно ли редактирование группе пользователя?
- да => редактирование нити доступно, конец
- нет => редактирование нити не доступно, конец
- нет такого правила => ошибка в таблице acl, не заданы разрешения по умолчанию для группы
Наличе прав на редактирование автоматически означает наличие прав на просмотр. Права на модерирование нити (редактирование
настроек) проверяются аналогично.
